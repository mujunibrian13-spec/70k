ERRORS: Undefined variables: can_save and next_save_date in savings.php
STATUS: ✅ FIXED

================================================================================
WHAT WAS WRONG
================================================================================

The variables $can_save and $next_save_date were only defined inside an 
if/else block, but were being used outside of that block.

PROBLEM:
  if ($member_id === 0) {
      // Variables NOT initialized here
  } else {
      $can_save = true;           // Only initialized here
      $next_save_date = date();   // Only initialized here
  }
  
  // ERROR: Using variables that might not be defined!
  if (!$can_save) {  // ❌ Undefined if member_id === 0!
      $warning = '...';
  }

================================================================================
HOW IT WAS FIXED
================================================================================

1. INITIALIZE ALL VARIABLES FIRST
   Before the if/else block, initialize all variables with default values:
   
   $member = null;
   $current_savings = 0;
   $week_savings = null;
   $can_save = true;              // Default: can save
   $next_save_date = date('Y-m-d'); // Default: today

2. OVERRIDE IN IF/ELSE BLOCKS
   Then in the else block, override them with actual values:
   
   if ($member_id === 0) {
       // No changes needed, use defaults
   } else {
       $can_save = !$week_savings;  // Override
       $next_save_date = getNextSavingsDate(...); // Override
   }

3. USE SAFELY OUTSIDE
   Now variables are always defined:
   
   if (!$can_save && $week_savings) {  // ✅ Always defined!
       $warning = '...';
   }

================================================================================
ERROR PREVENTION PATTERN
================================================================================

Best Practice for Conditional Variable Initialization:

// GOOD - Always do this:
$var1 = default_value_1;
$var2 = default_value_2;
if (condition) {
    $var1 = new_value_1;  // Override if needed
    $var2 = new_value_2;
}
// Safe to use $var1 and $var2 here

// BAD - Avoid this:
if (condition) {
    $var1 = value_1;  // Only initialized here
    $var2 = value_2;
}
// Unsafe to use $var1 and $var2 here (might not be defined)
