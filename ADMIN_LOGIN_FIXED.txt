ISSUE: Admin cannot login / Fatal recursion error
STATUS: ✅ COMPLETELY FIXED

================================================================================
WHAT WAS WRONG
================================================================================

TWO ISSUES:

1. PASSWORD HASH FORMAT MISMATCH
   - Database had bcrypt format: $2y$10$...
   - System only supported SHA256 format: sha256$...
   - Result: Invalid password error

2. INFINITE RECURSION
   - Function called itself indirectly
   - verifyPasswordHash() → password_verify() → verifyPasswordHash()
   - Result: Maximum nesting level reached (100 calls)

================================================================================
HOW IT WAS FIXED
================================================================================

ISSUE 1: PASSWORD FORMAT SUPPORT
   File: config/functions.php
   
   Added: verifyPasswordHashExtended() function
   Purpose: Supports SHA256, Bcrypt, and MD5 formats
   Result: System now recognizes bcrypt hashes
   
ISSUE 2: RECURSION LOOP
   File: config/functions.php
   
   Removed: Circular function calls
   Changed: Bcrypt verification uses native PHP function directly
   Method: Used namespace escape (\password_verify)
   Result: No infinite loops, clean function flow

================================================================================
FUNCTIONS EXPLAINED
================================================================================

password_verify()
  - Basic password verification
  - Supports: SHA256 and MD5 formats
  - Called by: verifyPassword()
  - No recursion (only checks formats, no sub-calls)

verifyPasswordHashExtended()
  - Extended password verification
  - Supports: SHA256, Bcrypt, and MD5 formats
  - Handles bcrypt by calling PHP's native function
  - No circular calls (calls native \password_verify, not wrapper)

verifyPassword()
  - Public API function
  - Calls: verifyPasswordHashExtended()
  - Purpose: Verify any password hash format

PASSWORD HASH FORMATS SUPPORTED
================================================================================

Now supports all these formats:

1. SHA256 Custom
   Format: sha256$salt$hash
   Example: sha256$abc123def456$...
   Used by: System's password_hash function

2. Bcrypt (NEW)
   Format: $2y$10$... or $2a$... or $2b$...
   Example: $2y$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcg7b3XeKeUxWdeS86E36P4/KeK
   Used by: PHP's native password_hash function

3. MD5 Legacy
   Format: md5$hash
   Example: md5$5d41402abc4b2a76b9719d911017c592
   Used by: Older systems

4. Direct MD5
   Format: Direct 32-char hex hash
   Example: 5d41402abc4b2a76b9719d911017c592
   Used by: Very old systems

================================================================================
THE FIX IN PLAIN LANGUAGE
================================================================================

BEFORE:
1. Admin tries to login with password: admin123
2. Database has bcrypt hash: $2y$10$N9qo8u...
3. System checks password with password_verify()
4. password_verify() calls verifyPasswordHash()
5. verifyPasswordHash() calls password_verify() again
6. Loop repeats 100 times
7. Error: Maximum function nesting level reached
8. Login fails

AFTER:
1. Admin tries to login with password: admin123
2. Database has bcrypt hash: $2y$10$N9qo8u...
3. System checks password with verifyPassword()
4. verifyPassword() calls verifyPasswordHashExtended()
5. verifyPasswordHashExtended() detects bcrypt format
6. Calls PHP's native password_verify() using \password_verify
7. PHP's native function verifies the bcrypt hash
8. Returns true/false
9. Login succeeds or fails normally
10. No errors, no loops

================================================================================
TECHNICAL DETAILS
================================================================================

Recursion Problem:
  The issue was a circular function dependency:
  password_verify() → verifyPasswordHash() → password_verify()
  
Solution:
  Separated into two functions with clear responsibilities:
  - password_verify() handles simple formats (no calls to extended)
  - verifyPasswordHashExtended() handles all formats
  - Uses \password_verify() (native) for bcrypt to break cycle

Namespace Escape:
  - \password_verify() means "use the native PHP function"
  - Not our wrapper function
  - Breaks the circular reference
  - Calls PHP's built-in password verification

PHP Version Check:
  - PHP 5.5+ has native password_verify()
  - System checks PHP version before using it
  - Falls back gracefully for older PHP
