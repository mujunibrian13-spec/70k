                    ADMIN LOGIN REDIRECT LOOP - FIXED
================================================================================

PROBLEM:
  When trying to login as admin, Firefox shows:
  "The page isn't redirecting properly"
  "The server is redirecting the request in a way that will never complete"

ROOT CAUSE:
  The admin user in the database doesn't have the 'admin' role set,
  causing the isAdmin() function to return false, which redirects
  back to login.php, creating an infinite redirect loop.

SOLUTION:
  Use the new fix_admin_login.php page to automatically fix the admin user.

================================================================================
                    HOW THE REDIRECT LOOP HAPPENS
================================================================================

The Loop:

1. User goes to login.php
2. User enters admin credentials
3. Login.php checks password (succeeds)
4. Login.php sets session variables
5. Login.php redirects to admin.php
6. Admin.php checks isLoggedIn() → TRUE (session exists)
7. Admin.php checks isAdmin() → FALSE (role not set to 'admin')
8. Admin.php redirects to login.php
9. Browser sees redirect and tries login.php again
10. Infinite loop! Browser gives up and shows error

The Fix:

Step 7 now returns TRUE because role is properly set to 'admin',
so admin.php loads successfully instead of redirecting back.
